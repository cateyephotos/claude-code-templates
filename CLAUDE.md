# CLAUDE.md

This file provides guidance to Claude Code when working with the SupplementDB repository.

## Project Overview

SupplementDB is an evidence-based supplement database at **supplementdb.info**. The codebase lives in `supp-db-site/` and consists of:

- **Static frontend** (vanilla HTML/CSS/JS) deployed to **Vercel**
- **Convex backend** (TypeScript actions/queries/mutations) deployed separately
- **Admin dashboard** at `/admin/` with analytics panels (GA4, PostHog, GSC, self-hosted)
- **Supplement monograph pages** in `supplements-new/` (generated by research pipeline)
- **Anti-scraping honeypots** injected into all HTML pages via `scripts/inject-honeypots.js`

## Critical: Two Convex Deployments

SupplementDB has two independent Convex deployments. **Getting this wrong causes silent failures.**

| Environment | Deployment | `.env.local` default |
|---|---|---|
| Local dev | `robust-frog-754` | Yes (CONVEX_DEPLOYMENT in .env.local) |
| **Production** | `acoustic-chinchilla-759` | No — requires `--prod` flag |

### The `--prod` Rule

**`npx convex env set` defaults to the dev deployment** (from `.env.local`). To set env vars on production:

```bash
# CORRECT — targets prod
npx convex env set --prod GA4_PROPERTY_ID="530443869"

# WRONG — goes to dev, production never sees it
npx convex env set GA4_PROPERTY_ID="530443869"
```

The same applies to `npx convex env list`:
```bash
npx convex env list --prod   # See what production actually has
npx convex env list           # Shows dev deployment vars
```

**`npx convex deploy`** always targets prod (this is correct). No `--prod` flag needed for deploy.

### Deploy Commands

```bash
cd supp-db-site

# Deploy Convex backend — MUST use this script, never npx convex deploy directly
npm run deploy:convex:prod

# Deploy Vercel frontend
vercel --prod
```

**NEVER run `npx convex deploy` directly.** The `auth.config.js` file defaults to the dev Clerk domain (`usable-tarpon-30.clerk.accounts.dev`). Production requires `clerk.supplementdb.info`. The `deploy:convex:prod` script handles the domain swap automatically — it swaps to the prod domain, deploys, then reverts for local dev. Deploying without this script silently breaks all authenticated Convex queries (admin dashboard, user-specific data) because the JWT issuer won't match the configured auth provider.

TypeScript typecheck is disabled because `gsc.ts` and `ga4.ts` have a known TS2802 error (Uint8Array iteration without downlevelIteration) that doesn't affect Convex runtime.

## Environment Variables

### Convex Prod Env Vars (source of truth: Convex Dashboard)

Per ADR 001 (`docs/adr/001-convex-env-management.md`), the Convex dashboard is the source of truth for server-side secrets. Doppler manages app-layer secrets only (Vercel, CLI tooling).

Key env vars on prod (`acoustic-chinchilla-759`):

| Var | Purpose | Notes |
|-----|---------|-------|
| `GSC_SERVICE_ACCOUNT_JSON` | Google service account for GSC + GA4 API | Single-line JSON string. SA email: `supplementdb-analytics@supplementdb-492502.iam.gserviceaccount.com` |
| `GSC_SITE_URL` | Site URL for GSC queries | `https://supplementdb.info` |
| `GA4_PROPERTY_ID` | GA4 numeric property ID | `530443869` (NOT the G- measurement ID) |
| `POSTHOG_API_KEY` | PostHog personal API key (`phx_...`) | NOT the client capture key (`phc_...`). Must use `/query/` endpoint. |
| `POSTHOG_HOST` | PostHog API host | `https://us.posthog.com` (NOT `us.i.posthog.com` which is ingestion-only) |
| `POSTHOG_PROJECT_ID` | PostHog project ID | `325670` |

### PostHog Key Types

- `phc_...` = **client capture key** (browser-side, used by posthog-js to send events)
- `phx_...` = **personal API key** (server-side, used by Convex actions to query data)

Using the wrong key type causes silent 401/403 failures.

## Code Patterns

### Security: DOM Construction

A pre-commit hook rejects setting element `.innerHTML` with untrusted content. All DOM construction in admin panel JS must use safe DOM methods:

```javascript
// CORRECT — safe DOM construction
const el = document.createElement('div');
el.textContent = userInput;
parent.appendChild(el);
```

Never set raw HTML strings on elements. Use `createElement`, `textContent`, and `appendChild` instead.

### PostHog API: Use HogQL Query Endpoint

PostHog deprecated legacy `/insights/trend/` and `/insights/funnel/` endpoints. All PostHog Convex actions must use the `/query/` endpoint with HogQL:

```typescript
// CORRECT — HogQL via /query/
const data = await posthogRequest(host, `/api/projects/${projectId}/query/`, apiKey, {
  method: "POST",
  body: {
    query: {
      kind: "HogQLQuery",
      query: "SELECT properties.$referring_domain, count() FROM events WHERE event = '$pageview' GROUP BY 1 ORDER BY 2 DESC LIMIT 10"
    }
  }
});
```

The legacy `/insights/trend/` endpoint returns 403 "Legacy insight endpoints are not available for this user."

### Google APIs: Service Account JWT Flow

Both `convex/gsc.ts` and `convex/ga4.ts` use the same pattern:
1. Parse `GSC_SERVICE_ACCOUNT_JSON` from env
2. Create JWT signed with the service account's private key
3. Exchange JWT for OAuth2 access token at `oauth2.googleapis.com/token`
4. Call the Google API with the access token

The service account needs:
- **GSC**: Added as Full user in Search Console → Settings → Users
- **GA4**: Added as Viewer in GA4 → Admin → Property Access Management

### Traffic Data Hygiene (P0)

Client-side `convex-client.js` has:
- `shouldSkipCapture()` — skips recording page views from non-prod hosts (localhost, vercel.app, etc.)
- `sanitizeReferrer()` — drops self-referrals from supplementdb.info

Server-side `metrics.ts` has:
- `categorizeReferrer()` — buckets referrers into: organic, social, referral, internal, auth, direct
- `getTrafficSources()` with `cleanOnly` flag — drops internal/auth categories from admin view

## Common Gotchas

1. **Convex deploy MUST use `npm run deploy:convex:prod`** — Never `npx convex deploy` directly. The auth.config.js has the dev Clerk domain by default; the deploy script swaps it to prod, deploys, then reverts. Skipping this silently breaks all authenticated queries (admin dashboard shows skeleton loading, "Authentication required" in Convex logs). Symptom is invisible — no browser console errors, Clerk shows signed in, but Convex rejects the JWT because the issuer domain doesn't match.
2. **Convex env vars on wrong deployment** — Always use `--prod` for production env vars
3. **PostHog phc_ vs phx_ keys** — Server-side actions need `phx_` personal API keys
4. **PostHog host** — API queries go to `us.posthog.com`, event ingestion goes to `us.i.posthog.com`
5. **PostHog legacy endpoints** — `/insights/trend/` is deprecated, use `/query/` with HogQL
6. **DOM security hook** — Pre-commit hook blocks unsafe HTML injection; use safe DOM methods
7. **Vercel env vars** — Never use echo to pipe; always trim in build scripts
8. **Vercel cleanUrls** — Breaks relative paths in subdirectories
9. **GA4 property ID** — Use the numeric ID (530443869), not the G- measurement ID
10. **GSC property owner** — Property is on `carlostomasphotos@gmail.com`, not `carlosthomas.01@gmail.com`
11. **TypeScript typecheck** — Disable for deploy (`--typecheck=disable`) due to known TS2802 in gsc.ts/ga4.ts
