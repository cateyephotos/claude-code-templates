Accessibility Statement

Last Updated: March 8, 2026

1. Our Commitment

SupplementDB is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards to make our supplement research database usable by the widest possible audience.

We believe that access to evidence-based health information should be available to all individuals, regardless of ability.

In plain language: We want everyone to be able to use SupplementDB, including people who use screen readers, keyboard navigation, or other assistive technologies. We're actively working to make that possible.

2. Conformance Standard

We aim to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standard. These guidelines explain how to make web content more accessible to people with a wide range of disabilities, including:

  • Visual impairments (blindness, low vision, color blindness)
  • Hearing impairments
  • Motor impairments
  • Cognitive and learning disabilities

WCAG 2.1 Level AA is also referenced in various legal frameworks, including the Americans with Disabilities Act (ADA), Section 508 of the Rehabilitation Act, and the European Accessibility Act.

3. Accessibility Features

SupplementDB incorporates the following accessibility features:

3.1 Navigation and Structure

  • Semantic HTML — We use proper heading hierarchy (h1–h6), landmark regions, and semantic elements to ensure screen readers can accurately convey page structure
  • Keyboard navigation — All interactive elements (links, buttons, search, filters, modals) are accessible via keyboard using Tab, Enter, and Escape keys
  • Skip navigation — Users can bypass repetitive content to access main page content directly
  • Consistent navigation — Navigation patterns are consistent across all pages

3.2 Visual Design

  • Color contrast — Text and interactive elements meet WCAG 2.1 AA contrast ratio requirements (minimum 4.5:1 for normal text, 3:1 for large text)
  • Non-color indicators — Information is not conveyed by color alone; our evidence tier system uses text labels alongside color coding
  • Responsive design — The Website is fully responsive and can be zoomed up to 200% without loss of content or functionality
  • Readable typography — We use legible fonts (Source Serif 4, DM Sans) with adequate line spacing (1.5+) and font sizes

3.3 Content Accessibility

  • Plain language annotations — Legal documents include plain-language summaries marked with distinctive styling for easy identification
  • Descriptive links — Link text describes the destination or action, avoiding generic phrases like "click here"
  • ARIA labels — Interactive components include ARIA attributes where native HTML semantics are insufficient
  • Focus indicators — Visible focus outlines are present on all interactive elements for keyboard users

3.4 Data Presentation

  • Structured data tables — Supplement data is presented in properly marked-up tables with headers and scope attributes
  • Alternative text — Icons include appropriate aria-hidden or aria-label attributes to ensure meaningful communication to screen reader users
  • Consistent formatting — Dosage information, evidence ratings, and safety data follow consistent patterns for predictable reading

4. Known Limitations

While we strive for full compliance, we acknowledge the following current limitations:

  • Third-party content — External links to PubMed and DOI-registered publications lead to websites outside our control; their accessibility levels may vary
  • Complex data visualizations — Some evidence tier visual indicators may not fully convey information to screen readers; text alternatives are provided alongside visual elements
  • Dynamic content — Some dynamically loaded content (search results, filter changes) may not always announce updates to screen readers immediately; we are working to improve ARIA live region implementation
  • PDF exports — If PDF export features are added in the future, initial versions may have limited accessibility until remediation is complete

We are actively working to address these limitations and improve accessibility across the entire Website.

5. Assistive Technology Compatibility

SupplementDB has been designed to be compatible with the following assistive technologies:

  • Screen readers — NVDA (Windows), VoiceOver (macOS/iOS), TalkBack (Android)
  • Voice recognition — Dragon NaturallySpeaking, Windows Speech Recognition
  • Screen magnification — Windows Magnifier, macOS Zoom, browser zoom (up to 200%)
  • Keyboard-only navigation — Full site functionality available without a mouse
  • High contrast modes — Compatible with Windows High Contrast mode and browser high-contrast extensions

6. Testing and Evaluation

We evaluate the accessibility of our Website through:

  • Automated accessibility testing tools (Lighthouse, axe DevTools, WAVE)
  • Manual keyboard navigation testing
  • Screen reader testing with NVDA and VoiceOver
  • Color contrast analysis tools
  • Responsive design testing across device sizes

We conduct accessibility reviews when significant content or design changes are made.

7. Feedback and Accommodation Requests

We welcome your feedback on the accessibility of SupplementDB. If you encounter accessibility barriers, need information in an alternative format, or have suggestions for improvement, please contact us:

legal@supplementdb.com

Subject line: Accessibility Feedback

When contacting us, please include:

  • The page URL where you encountered the issue
  • A description of the accessibility barrier
  • The assistive technology you use (if applicable)
  • Your preferred contact method for follow-up

We will acknowledge your feedback within 5 business days and work to resolve accessibility issues as promptly as possible.

Alternative Access

If you are unable to access specific supplement information on our Website due to an accessibility barrier, please contact us and we will provide the information in an accessible alternative format at no cost.

8. Continuous Improvement

Accessibility is an ongoing effort. Our improvement roadmap includes:

  • Implementing enhanced ARIA live regions for dynamic content updates
  • Adding skip-to-content links on all pages
  • Conducting formal WCAG 2.1 AA audit with third-party accessibility specialists
  • Incorporating user feedback into design and development processes
  • Training our content team on accessible writing practices
  • Regular testing with users who rely on assistive technologies
