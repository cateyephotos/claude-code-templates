Stack Analyzer UX Comparison

Three approaches to the supplement selection flow. Click each tab to explore the interactive mockup, then review the comparison summary.

A

Current: Supplements First

Users select supplements from a category-sorted dropdown, then pick a health goal, then analysis depth. The current production flow.

Strengths

  • Works well for users who already know their stack
  • No cognitive reframing needed — direct supplement lookup
  • Simple, linear flow that's easy to understand

Weaknesses

  • 93 supplements in a flat alphabetical list is overwhelming
  • No guidance for exploratory users ("I want better sleep")
  • Category grouping doesn't reflect goal relevance
  • Health goal selection feels disconnected from supplement choices

B

Goal-First with Smart Grouping

Users first select their health goal, then supplements are dynamically grouped by domain relevance using problems.js relevanceScore data. High-relevance supplements surface to the top.

Strengths

  • Guides exploratory users toward relevant supplements
  • Relevance scores provide evidence-based ranking
  • Domain grouping reduces cognitive load (from 93 flat to chunked)
  • Health goal context improves AI analysis quality

Weaknesses

  • Forces goal selection even for users who just want to check a stack
  • May bias users toward "popular" supplements over their actual stack
  • Extra step before supplement selection

C

Hybrid Adaptive Flow

Both paths are available. An optional health goal toggle reorders and groups supplements by relevance when activated. Without it, the list shows the familiar category grouping. Serves both user types seamlessly.

Strengths

  • Serves both experienced and exploratory users
  • No forced steps — goal is optional but impactful
  • Smooth transition: toggling goal re-sorts in real time
  • Retains familiarity while adding smart grouping as an enhancement

Weaknesses

  • Slightly more complex UI (toggle + conditional layout)
  • Users may not discover the goal toggle without guidance
  • Two modes need to be maintained

Recommendation: Approach C (Hybrid Adaptive)

Approach C preserves the direct-access path that experienced users expect while layering in the smart relevance grouping from Approach B as an opt-in enhancement. The "Smart Sort by Health Goal" toggle is prominent enough for exploratory users to discover, yet unobtrusive enough that power users aren't slowed down. It also reduces the step count from 3 to 2 (supplements + depth), folding the goal into an optional context-setter rather than a mandatory gate.

The problems.js relevanceScore data (0-100 per supplement per domain) is already available and makes the grouping data-driven rather than arbitrary. Implementation adds a toggle state and conditional sort/render logic to the existing supplement dropdown — a moderate lift with high UX payoff for both user types.
